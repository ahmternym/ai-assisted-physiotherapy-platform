**Tarih:** 2026-03-31

## 🐛 Giderilen Hatalar

### BUG-1: Harita Gözükmüyor

**Kök neden:** `frontend/index.html` yanlış konumdaydı.

Vue CLI şu yapıyı bekler:
```
frontend/
  public/
    index.html   ← burada olmalı
```

Mevcut durumda dosya `frontend/index.html` olarak duruyordu. Vue CLI bu dosyayı görmez, kendi boş HTML şablonunu kullanır ve içinde Leaflet CDN yoktur.

**İkincil neden:** `MapSelector.vue`'da `mounted()` hook'u içinde `if (!window.L) return` yazılıydı. CDN yavaş yüklenirse `window.L` henüz hazır olmaz ve harita tamamen atlanırdı.

**Yapılan değişiklikler:**
- `frontend/public/index.html` oluşturuldu (Vue CLI'nin beklediği konum)
- Leaflet CSS `<head>`'de, JS `</body>` öncesinde yükleniyor
- `MapSelector.vue`'da `tryInitMap(attempt)` retry fonksiyonu eklendi:
  ```js
  tryInitMap(attempt) {
    if (window.L) { this.initMap(); return }
    if (attempt >= 10) { this.mapError = true; return }
    setTimeout(() => this.tryInitMap(attempt + 1), 300)
  }
  ```
- Harita başlatamazsa `map-fallback` div gösteriliyor
- `vue.config.js`'e `indexPath: 'index.html'` eklendi

### BUG-2: 81 İl Çalışmıyor (sadece popüler iller)

**Kök neden:** `api.js`'de hardcoded tam URL kullanılıyordu:
```js
const BASE_URL = 'http://localhost:8080/api'  // YANLIŞ
```

`vue.config.js` proxy'si sadece göreli URL'ler (`/api/...`) için devreye girer. Tam URL yazıldığında tarayıcı isteği doğrudan backend'e atar. Bu durumda:
- Backend `localhost:8080` üzerinde çalışıyor
- Frontend `localhost:5173` üzerinde çalışıyor
- Farklı port = farklı origin = CORS hatası
- Tarayıcı isteği **bloklar**

Popüler iller de aslında çalışmıyordu — sadece backend ve frontend aynı makinede ve CORS header'ı backend tarafında `*` olarak ayarlandığı için bazı durumlarda geçti.

**Yapılan değişiklik:**
```js
// YENİ — api.js
const api = axios.create({ baseURL: '/api' })  // DOĞRU — proxy çalışır
```

### BUG-3: Kartlara Tıklayınca Detay Açılmıyor

**Kök neden:** Vue 3'te `$emit('click', value)` kullanımı native DOM click ile çakışır.

`PhysiotherapistCard.vue`:
```html
<div @click="$emit('click', physio.id)">  <!-- YANLIŞ -->
```

`List.vue`:
```html
<PhysiotherapistCard @click="goToDetail" />  <!-- @click native DOM'u dinler -->
```

Vue 3'te `@click` bir bileşen üzerinde kullanıldığında **native click eventi** dinlenir, custom emit değil. `goToDetail` fonksiyonuna `physio.id` yerine `MouseEvent` objesi geliyordu. `$router.push({ params: { id: MouseEvent } })` çalışmıyor, detay sayfası açılmıyordu.

**Yapılan değişiklikler:**

`PhysiotherapistCard.vue`:
```js
emits: ['select-physio'],  // Vue 3: emits açıkça tanımla
methods: {
  onCardClick() {
    this.$emit('select-physio', this.physio.id)  // native 'click' değil
  }
}
```

`List.vue`:
```html
<!-- @click yerine özel event adı -->
<PhysiotherapistCard @select-physio="goToDetail" />
```

### BUG-4: Uzmanlık Alanları Eksik (Kısmi)

**Durum:** Backend `data.go`'daki 405 kayıtta `Specialties []string` alanı mevcut ve dolu. Hiçbir kayıtta boş bırakılmamış. Ancak önceki sürümlerde bazı iller için tek uzmanlık girilmişti.

**Mevcut durum:** Tüm kayıtlar 8 uzmanlık kategorisinden en az 1-2 tanesine sahip. Detay sayfasında `v-if="physio.specialties && physio.specialties.length"` kontrolü var — boş gelirse bölüm gizleniyor.
